# Prototype

Build the smallest artifact that answers the open question.

Rules:

- State what the prototype is meant to prove or disprove.
- Mark whether it is throwaway, reference-only, or production-bound.
- For UI work, create comparable variants when that helps decision-making.
- For logic or state work, keep the prototype small and isolated.
- Record the result, decision, and what should or should not be reused.

Do not silently turn a prototype into production code without review.
